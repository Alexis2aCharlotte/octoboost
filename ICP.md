## 1. ICP principal : Le Vibe Coder qui lance son SaaS

### Qui il est

| Attribut | Detail |
|----------|--------|
| **Age** | 20-35 ans |
| **Profil technique** | Non-developpeur ou developpeur junior. Utilise Cursor, Bolt, Replit, Lovable, v0 pour builder |
| **Background** | Ex-freelance, ex-marketeur, ex-designer, etudiant, reconversion. Pas sorti d'ecole d'ingenieur |
| **Projet** | Micro-SaaS, side project, outil de niche, app AI. Souvent son premier produit |
| **Stack** | Next.js / React (vibe-code), deploye sur Vercel ou Netlify. Parfois Astro, Hugo |
| **Revenue** | Pre-revenue a $1-5K MRR. Bootstrap, pas de levee |
| **Budget outils** | $50-150/mois total (Cursor $20 + Vercel $20 + domaine + divers) |
| **Langue** | Anglais (marche global, US + EU + monde) |
| **Ou il traine** | Twitter/X, YouTube (tutos vibe coding), TikTok, Reddit (r/SaaS, r/indiehackers), Discord |

### Ce qu'il sait faire

- Builder un produit fonctionnel avec l'IA (Cursor, Bolt)
- Deployer sur Vercel en un clic
- Utiliser GitHub (push, PR, branches)
- Configurer un domaine, Supabase, Stripe
- Suivre des tutos et adapter du code

### Ce qu'il ne sait PAS faire

- Keyword research (ne sait pas par ou commencer)
- Ecrire des articles SEO de 2000+ mots optimises
- Comprendre les canonical URLs, le maillage interne, les balises meta
- Cross-poster sur Dev.to, Hashnode, Medium avec le bon format
- Mettre en place une strategie de contenu long terme
- Configurer Google Search Console et interpreter les donnees
- Eviter les 17 erreurs SEO classiques du vibe coding (client-side rendering, pas de SSR, mauvaise structure URL, meta tags manquants)

### Son etat emotionnel

- **Frustre** : il a build un bon produit mais personne ne le trouve sur Google
- **Impatient** : il veut des resultats, pas passer 3 mois a apprendre le SEO
- **Sous pression** : il sait que sans trafic organique, il reste dependant des ads payantes
- **Pret a payer** : il paye deja $20/mois pour Cursor sans hesiter. $19/mois pour du trafic gratuit = no-brainer
- **Overwhelmed** : il voit des threads Twitter "voici ma strategie SEO en 47 etapes" et se dit "jamais"

### Ses objectifs

1. Avoir ses premiers utilisateurs organiques (pas que du Product Hunt + Twitter)
2. Apparaitre sur Google quand quelqu'un cherche son type de produit
3. Etre cite par ChatGPT, Perplexity, Claude quand on pose une question dans sa niche
4. Automatiser son marketing pour se concentrer sur le produit
5. Reduire sa dependance aux ads payantes (Meta, Google Ads)

## 2. ICP secondaire (Phase 2, pas maintenant)

### Le freelance/consultant avec un site WordPress

| Attribut | Detail |
|----------|--------|
| **Profil** | Coach, consultant, freelance, createur de contenu |
| **Stack** | WordPress, Webflow, parfois Squarespace |
| **Budget** | $100-300/mois en outils |
| **Pain point** | Sait que le blog SEO attire des clients mais n'a pas le temps d'ecrire |

Ce profil arrive en Phase 2 quand l'integration WordPress site principal sera finalisee et que le messaging sera adapte. On ne le cible pas tant que l'ICP principal n'est pas valide et rentable.

### Pourquoi PAS le e-commerce / Shopify

| Raison | Detail |
|--------|--------|
| **Concurrence extreme** | Surfer SEO ($99-219/mo), Jasper ($49-125/mo), Semrush ($139-500/mo), Rankable, Sebora, AutoRank ciblent deja ce segment |
| **Effort produit** | Construire une app Shopify (OAuth, review process, Blog API) = 4-6 semaines minimum |
| **Mismatch storytelling** | L'histoire "builder for builders" ne resonne pas avec un dropshipper |
| **CAC trop eleve** | Le CPC ecomm sur Meta est $3-8, CPA ~$150+. A $19/mo le payback depasse 8 mois |
| **Churn eleve** | Les owners Shopify testent 10 apps, gardent 2. Retention faible sur les outils SEO low-ticket |

### Unit economics cibles

| Metrique | Cible | Seuil d'alerte |
|----------|-------|----------------|
| CPA (user payant) | $60-100 | >$120 = non rentable |

## 8. Regles de decision produit

Avant d'ajouter une feature, poser ces questions :

1. **Est-ce que mon ICP (vibe coder non-technique qui lance un micro-SaaS) en a besoin ?**
   - Si non → on ne le fait pas.

2. **Est-ce que ca simplifie ou ca complexifie l'experience ?**
   - Si ca ajoute du jargon technique ou une etape manuelle → redesigner.

3. **Est-ce que ca ameliore le "wow moment" (premiere analyse + premier article) ?**
   - Si oui → priorite haute.

4. **Est-ce que ca ameliore la retention (le user voit les resultats de son SEO) ?**
   - Si oui → priorite haute.

5. **Est-ce que ca ouvre un nouveau segment (ecomm, agences) avant d'avoir valide l'ICP principal ?**
   - Si oui → on attend. Valider l'ICP principal d'abord.

6. **Est-ce que mon concurrent le fait et que c'est un table-stakes ?**
   - Verifier Sebora, Rankable, AutoRank. Si oui, evaluer.

## 9. Metriques de validation ICP

L'ICP est valide quand :

| Signal | Seuil |
|--------|-------|
| Free → Paid conversion rate | >15% |
| Churn M1 | <15% |
| NPS (Net Promoter Score) | >40 |
| % users qui publient >3 articles | >50% |
| Ratio LTV/CAC sur Google Ads | >3x |
| Nombre de users "organiques" (pas ads) | En croissance mois/mois |
| Reviews/testimonials spontanes | >5 en 3 mois |

Si ces seuils ne sont pas atteints en 3-6 mois, remettre en question l'ICP et pivoter vers l'ICP secondaire (freelances/consultants WordPress).
